How do scenes get from Studio → Sim?

This is the central design decision — everything else follows from it.

A

File-based

Studio saves scenes as JSON files. Sim watches the filesystem and loads them on demand. Simple, durable, no live sync.

Studio → writes → scenes/solar-system.json
Sim → reads → file on load / manual refresh
MCP → reads same files → exposes via tools
B

Live WebSocket sync

Studio pushes scene changes in real time through the MCP server's WebSocket to Sim. No files — the scene lives in memory.

Studio → WS push → MCP server → WS broadcast → Sim
Changes appear instantly with no manual step
MCP / Claude can also push scenes the same way
C

Hybrid: files + live push

Studio saves to JSON (durable). Also has a "push to sim" button that sends the current scene over WebSocket for instant preview. Best of both.

Studio → saves → scenes/*.json (persistent)
Studio → "▶ Run in Sim" → WS push (live preview)
MCP → can load any saved scene by name
