Layer 2: The Skill Suite

4 focused skills — each loads only when relevant, each has one clear job

🏗️
scene-builder
Loads when: building or modifying any 3D environment
Contains
  • Proven Three.js patterns (PBR planet, atmosphere shader, star field, bloom)
  • Public texture sources (NASA, Solar System Scope URLs)
  • Cesium integration patterns (realistic Earth + Three.js overlay)
  • Lighting rigs for space (ambient + point sun, HDR env)
  • Camera control patterns (orbit controls, cinematic moves)
Aesthetic guide
  • Dark deep-space background (#000005)
  • Physically correct planet scale ratios
  • Glowing sun with lens flare + corona
  • Orbit paths as faint ellipses, not solid lines
  • Educational labels: always present, never cluttered
🔭
visual-reviewer
Loads when: any visual change is made — the autonomous review loop
Defines the exact Chrome MCP loop: navigate → screenshot → score against criteria → fix → repeat.
Evaluation checklist Claude runs on every screenshot: does it look like space? are labels readable? are controls responsive? is the sun the brightest thing? are planet sizes roughly proportional? is there something interesting to interact with? Only ships when all pass.
🪐
orbital-mechanics
Loads when: scripting planet positions, velocities, orbital params
A physics reference skill. Contains real data for all 8 planets + major moons: semi-major axis, eccentricity, inclination, mass, radius, orbital period, color/texture. Includes Kepler's laws as formulas Claude can apply directly to compute positions. So "put Mars at its correct position for April 2026" is a single instruction — Claude looks up the data, calculates, applies.
🎛️
scene-scripter
Loads when: controlling the sim programmatically via MCP
Documents every MCP tool with exact parameter shapes. Teaches Claude to script precise scenarios: "show Jupiter at opposition with its 4 Galilean moons, time-accelerated at 100x, with labels showing distance from Earth." One instruction → Claude calls the right MCP tools in the right order, no hand-holding.
How skills load: Each skill has a description field Claude reads at session start. "Build a solar system scene" → Claude sees scene-builder matches → loads it. It never loads all 4 at once. Small context cost, big capability gain.
